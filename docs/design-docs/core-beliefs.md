# 핵심 원칙

## 원칙 1: 계약 우선
내부 연동 API (`/internal/v1/*`)는 제품 핵심 경로이므로 명시적이고 버전 관리 가능한 계약으로 유지합니다.

## 원칙 2: 지속 컨텍스트는 git에 남긴다
한 번의 대화로 끝나지 않는 결정은 `docs/` 문서로 기록합니다.

## 원칙 3: 짧은 피드백 루프
범위가 큰 일괄 변경보다, 검증 가능한 작은 변경을 빠르게 반복합니다.

## 원칙 4: 중요한 규칙은 자동화한다
중요한 가드레일은 가능하면 CI에서 기계적으로 검사합니다.

## 원칙 5: 지도는 얇게 유지한다
`AGENTS.md`는 장문의 매뉴얼이 아니라 진입 지도 역할에 집중합니다.
