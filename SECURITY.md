# SECURITY

## 목표
개발 속도와 별개로 반드시 지켜야 하는 보안 최소선을 정의합니다.

## 기본 통제
- API 키/시크릿은 코드와 로그에 평문으로 남기지 않습니다.
- 내부 API는 인증/권한 경계를 명확히 유지합니다.
- 입력값 검증 실패는 일관된 오류 응답으로 처리합니다.

## 개발 규칙
- PR에서 민감정보 하드코딩 금지
- 외부 연동 시 실패/타임아웃/재시도 정책 명시
- 보안 관련 변경은 문서(`SECURITY.md`, API 계약) 동시 갱신

## 점검 항목
- [ ] 시크릿 노출 여부(코드/로그)
- [ ] 입력 검증 누락 여부
- [ ] 예외 메시지 과다 노출 여부
- [ ] 의존성 업데이트 시 보안 영향 검토
