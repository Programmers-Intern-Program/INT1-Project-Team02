# RELIABILITY

## 목표
에이전트 기반 변경 속도를 유지하면서도 서비스 안정성을 깨지 않도록 신뢰성 기준을 명시합니다.

## 기본 원칙
- 작은 PR을 빠르게 병합하고, 이상 징후는 즉시 후속 수정합니다.
- 실패는 숨기지 않고 재현 가능한 로그/테스트로 환원합니다.
- 변경 전후 검증 절차를 자동화하여 사람 의존도를 줄입니다.

## 신뢰성 체크리스트
- [ ] 모든 PR에서 `spotlessCheck`와 테스트 통과
- [ ] 내부 API 계약 변경 시 문서 동기화
- [ ] 예외 처리 경로(`GlobalExceptionHandler`) 회귀 확인
- [ ] 장애 시 롤백 또는 핫픽스 절차 확인

## 운영 지표(초기)
- 빌드 성공률
- 테스트 실패율
- PR 리드타임
- 회귀 수정 소요시간(MTTR)

## 개선 백로그
1. 핵심 API 경로에 통합 테스트 추가
2. 장애 재현 템플릿과 회고 포맷 문서화
3. 관측성(로그/메트릭) 기준 문서 추가
